Metric definitions & data sources
Weekly / Monthly Active Users (WAU / MAU)
Distinct non-admin user_ids with ≥1 event in the trailing 7 / 30 days. Source: usage_events — computable today, no new instrumentation.
WAU/MAU Stickiness
WAU ÷ MAU. The share of monthly customers who came back this week; >50% is strong for a monthly-cadence product. Source: usage_events — computable today.
Return Rate (within 30 days)
Of customers active in the prior 30-day window, the % with any event in the following 30 days. Needs 60 days of history — fits inside the 90-day retention purge. Source: usage_events — computable today.
Session Frequency
Sessions per active customer per week. A session is a burst of one user's events with no gap >30 minutes (derived client-side; login events undercount because magic-link sessions persist). Source: usage_events, derived — computable today; a session_id in metadata would make it exact.
Feature Depth
Distinct core actions per session, where core = catalog browse, reserve/cancel, my-list view, subscriptions view, arrivals view. Source: usage_events, derived from the same session clustering.
AOV & AOV Uplift
Average reserved basket value per customer for the catalog month, split subscribers vs à-la-carte. Uplift = subscriber AOV ÷ à-la-carte AOV − 1. Source: preorders × catalog.price_usd join (accurate, survives cancels); reserve events carry price_usd for trend approximations.
Win-back List (Inactive Customers)
Profiles with status = active and is_admin false, whose newest usage event is older than 30 days (or absent within the 90-day retention). Paper profiles (is_paper) never log in and are listed as invite-to-app candidates, not lapses. Source: user_profiles LEFT JOIN latest usage_events per user — computable today.
Admin filter
Excludes events whose user_id belongs to a profile with is_admin = true. Client-side set-filter against user_profiles — no schema change. Default ON. Note: admin self-actions ARE currently logged (only impersonation is skipped) — docs say otherwise; see F87 candidate.
