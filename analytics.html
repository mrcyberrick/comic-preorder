Metric definitions & data sources
Weekly / Monthly Active Users (WAU / MAU)
Distinct filtered user_ids with ≥1 event in the trailing 7 / 30 days. Source: usage_events.
WAU/MAU Stickiness
WAU ÷ MAU. The share of monthly customers who came back this week. Source: usage_events.
Return Rate (within 30 days)
Of customers active in the 30-day window ending 30 days ago, the % with any event in the following 30 days. Source: usage_events.
Session Frequency
Sessions per active customer per week, for the selected window. A session is a burst of one user's events with no gap >30 minutes. Source: usage_events, derived client-side.
Feature Depth
Distinct core actions per session, where core = catalog browse, reserve/cancel, my-list view, subscriptions view, arrivals view. Source: usage_events, derived from the same session clustering.
AOV & AOV Uplift
Average reserved basket value per customer for the current catalog month, split subscribers vs à-la-carte. Uplift = subscriber AOV ÷ à-la-carte AOV − 1. Source: preorders × catalog.price_usd.
Win-back List (Inactive Customers)
Profiles with status = active and is_admin false, whose newest usage event is older than 30 days (or absent within the 90-day retention). Paper profiles never log in and are listed as invite-to-app candidates, not lapses. Source: user_profiles + usage_events.
Admin filter
Excludes events whose user_id belongs to a profile with is_admin = true. Client-side set-filter — no schema change. Default ON. Note: admin self-actions ARE logged (only impersonated sessions are skipped).
Retention
usage_events are purged after 90 days by the monthly import. There is no "all time" view — every metric on this page is bounded by that window.
